# Bug 004: `ACTION` column missing from XLS export

## Summary

The export dialog states: *"Its format is compatible for an import into SquashTM."* However, the exported XLS file does not contain the `ACTION` column, which is **required** by SquashTM's import specification. Without an `ACTION` column, the exported file cannot be directly used as an import file, making the claim of import compatibility false.

## Steps to Reproduce

1. Navigate to any requirement folder in the Requirements workspace.
2. Click the import/export toolbar button and select **Export**.
3. In the dialog, select format **XLS**. Note the text: *"Its format is compatible for an import into SquashTM."*
4. Click **Export** and open the downloaded XLS file.
5. Examine the column headers of the **REQUIREMENT** sheet.

## Expected Result

The exported file should contain an `ACTION` column (as specified in the SquashTM import documentation), so that the file can be directly used as an import file after making minimal modifications.

## Actual Result

The `ACTION` column is absent. The exported columns are:

```
PROJECT_ID, PROJECT_NAME, REQ_PATH, REQ_NUM, REQ_ID, REQ_VERSION_NUM,
REQ_VERSION_ID, REQ_VERSION_REFERENCE, REQ_VERSION_NAME, REQ_NATURE,
REQ_VERSION_CRITICALITY, REQ_VERSION_CATEGORY, REQ_VERSION_STATUS,
REQ_VERSION_DESCRIPTION, REQ_VERSION_#_TC, REQ_VERSION_#_ATTACHEMENT,
REQ_VERSION_CREATED_ON, REQ_VERSION_CREATED_BY, REQ_VERSION_LAST_MODIFIED_ON,
REQ_VERSION_LAST_MODIFIED_BY
```

Additionally, the following import-spec columns are also missing:
- `ACTION` — required for import (e.g., values: `C` for Create, `U` for Update, `D` for Delete)

## Impact

The export-then-import round-trip workflow is broken. A user who:
1. Exports requirements to XLS to use as a template for updates
2. Modifies the exported file
3. Tries to import it back

...will find that the import fails or behaves unexpectedly because the `ACTION` column is missing.

The UI text claiming import compatibility is misleading.

## Environment

- SquashTM instance: http://host.docker.internal:8090/squash
- Export format: XLS
- Dialog text: "Its format is compatible for an import into SquashTM."
