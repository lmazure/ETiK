# Bug 001 — Folder path omitted from `REQ_PATH` in export

## Summary

The `REQ_PATH` column in the exported REQUIREMENT sheet omits intermediate folders from the path. Only the project name and the requirement name are included; any folders between them are silently dropped.

## Steps to Reproduce

1. Log in as `admin`.
2. In the Requirements workspace, open **TestProject**.
3. Verify the tree structure:
   - `TestProject`
     - `Authentication Module` *(folder)*
       - `REQ-001 User Authentication` *(requirement)*
4. Select **TestProject** in the tree and choose **Import/Export → Export**.
5. Click **Export** (default options).
6. Open the downloaded `.xls` file and inspect the `REQ_PATH` column in the `REQUIREMENT` sheet.

## Expected Result

```
/TestProject/Authentication Module/REQ-001 User Authentication
```

## Actual Result

```
/TestProject/REQ-001 User Authentication
```

The `Authentication Module` folder segment is missing.

## Impact

- **Data loss** — the exported file no longer encodes the folder hierarchy.
- **Broken round-trip** — importing the exported file places requirements directly under the project root instead of inside the `Authentication Module` folder, silently corrupting the structure.
- **Wrong paths in relationship tabs** — `LINK_REQ_TC`, `LINK_REQ_REQ`, and `LINK_HIGH_LEVEL_STANDARD_REQ` reference `REQ_PATH` values that will also be wrong, making relationship data unusable for re-import.

## Environment

- SquashTM instance: `http://host.docker.internal:8090/squash`
- Project: `TestProject`
- Requirement: `REQ-001 User Authentication` (inside folder `Authentication Module`)
