# Bug 010: Import log reports FAILURE for empty LINK_REQ_REQ and LINK_HIGH_LEVEL_STANDARD_REQ sheets

## Summary

When importing a requirements XLS file that was exported by SquashTM itself, the import log reports **FAILURE** for the `LINK_REQ_REQ` and `LINK_HIGH_LEVEL_STANDARD_REQ` sheets even when those sheets are validly present but empty (no data rows). The importer should handle empty sheets as a no-op. Since SquashTM Community edition always exports these sheets as empty (req-to-req linking requires a non-trivial UI interaction and HIGH_LEVEL requirements are a Premium feature), this error appears on every round-trip import of a Community edition export.

## Environment

- SquashTM Community Edition
- URL: `http://host.docker.internal:8090/squash`
- User: `admin`

## Steps to Reproduce

1. Export requirements from `TestProject` → `Special-Chars Test` folder to XLS (with "Keep the rich text format" unchecked to ensure all versions are exported — see Bug 005).
2. Confirm the exported XLS contains empty `LINK_REQ_REQ` and `LINK_HIGH_LEVEL_STANDARD_REQ` sheets (header row present, no data rows).
3. Import the exported XLS file via the Requirements workspace → toolbar → Import.
4. After import, download the import log XLS.
5. Examine the `LINK_REQ_REQ` and `LINK_HIGH_LEVEL_STANDARD_REQ` sheets in the log.

## Expected Result

Empty sheets exported by SquashTM are imported without error. The log shows no entries for `LINK_REQ_REQ` and `LINK_HIGH_LEVEL_STANDARD_REQ`, or shows them as "no data — skipped".

## Actual Result

The import log `LINK_REQ_REQ` sheet shows:

| LINE | TYPE | ERROR | IMPACT |
|------|------|-------|--------|
| 0 | FAILURE | The sheet is not present, present with an incorrect name or empty | |

And identically for `LINK_HIGH_LEVEL_STANDARD_REQ`.

The importer treats a validly-present but empty sheet the same as a missing or malformed sheet, and reports FAILURE.

## Impact

- Every round-trip import of a SquashTM Community edition export produces two spurious FAILURE entries in the import log, making it appear that data was lost.
- Users who read the log will investigate a non-existent problem.
- Automated import validation pipelines that check for any FAILURE will incorrectly flag every Community edition import as failed.

## Notes

- The FAILURE at LINE=0 is a structural/sheet-level error, not a row-level error. As a result, it does NOT appear in the import summary counts ("Associated requirements: 0 failed") — creating a discrepancy between the summary and the detailed log.
- Both affected sheets are always empty in Community edition exports:
  - `LINK_REQ_REQ`: req-to-req linking requires a drag-and-drop UI gesture that cannot be performed programmatically, making it difficult to populate in practice.
  - `LINK_HIGH_LEVEL_STANDARD_REQ`: HIGH_LEVEL requirements are a SquashTM Premium-only feature; they cannot be created in Community edition.
