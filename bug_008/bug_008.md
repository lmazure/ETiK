# Bug 008: Export truncates timestamps to date-only, discarding time information

## Summary

The `REQ_VERSION_CREATED_ON` and `REQ_VERSION_LAST_MODIFIED_ON` columns in the XLS export store only the **date** component (`YYYY-MM-DD`) of requirement version timestamps. The time-of-day portion is silently discarded. The SquashTM UI displays full timestamps (e.g. `6/5/26, 6:25 PM`), making this an information loss in the export.

## Environment

- SquashTM Community Edition
- URL: `http://host.docker.internal:8090/squash`
- User: `admin`

## Steps to Reproduce

1. Navigate to a requirement with a known creation/modification time (e.g. `REQ-SC01` in `TestProject/Special-Chars Test`).
2. Note the exact creation time displayed in the UI: `6/5/26, 6:25 PM (admin)`.
3. Export the folder to XLS (toolbar → import/export → Export).
4. Open the downloaded `.xls` file and examine the `REQ_VERSION_CREATED_ON` and `REQ_VERSION_LAST_MODIFIED_ON` columns.

## Expected Result

The exported value should include the time component to match the UI:

```
2026-06-05T18:25:00
```

or at minimum document the time in some way (e.g. `2026-06-05 18:25`).

## Actual Result

The exported value contains only the date with no time:

```
2026-06-05
```

This is identical whether using "Keep the rich text format" enabled or disabled.

## Impact

- **Audit trail degradation**: multiple requirement versions created on the same day cannot be distinguished by their creation timestamps in the export.
- **Round-trip fidelity**: re-importing the exported file will associate creation/modification dates with no time, losing the original precision.
- **Compliance risk**: regulated projects that need to track the exact time of requirement changes cannot rely on the export for audit purposes.

## Notes

- The column names use the suffix `_ON` (not `_DATE`), suggesting a full timestamp is intended.
- Both `REQ_VERSION_CREATED_ON` and `REQ_VERSION_LAST_MODIFIED_ON` are affected.
- The issue is reproducible with both "Keep the rich text format" enabled and disabled.
