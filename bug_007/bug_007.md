# Bug 007: Milestone order in export does not match the order displayed in the UI

## Summary

The `REQ_VERSION_MILESTONE` column in the XLS export lists milestones in a different order than they appear in the SquashTM UI. The UI displays milestones in the order they were associated (ascending, e.g. `Sprint 1, Sprint 2`), while the export outputs them in reverse order (e.g. `Sprint 2|Sprint 1`). This causes confusion and can break import-side tooling that relies on the exported milestone list.

## Environment

- SquashTM Community Edition
- URL: `http://host.docker.internal:8090/squash`
- User: `admin`

## Steps to Reproduce

1. Navigate to `TestProject` → `Special-Chars Test` folder → `REQ-SC01 Special <>&"' chars`.
2. In the **Version management** panel, observe Version 1 milestones: **Sprint 1, Sprint 2** (Spring 1 listed first).
3. Export the folder to XLS (toolbar → import/export → Export).
4. Open the exported `.xls` file and inspect the `REQ_VERSION_MILESTONE` cell for `REQ-SC01`.

## Expected Result

The exported milestone list matches the UI order:

```
Sprint 1|Sprint 2
```

## Actual Result

The exported milestone list is in reverse order relative to the UI:

```
Sprint 2|Sprint 1
```

## Evidence

**UI (Version management panel, Version 1 row):**

| Version nb | Milestones | TC nb |
|------------|------------|-------|
| 1 | Sprint 1, Sprint 2 | 0 |

**Export (`REQ_VERSION_MILESTONE` column, both export modes):**

```
Sprint 2|Sprint 1
```

The discrepancy is reproducible with both "Keep the rich text format" enabled and disabled.

## Impact

- Users who read milestones from the export for auditing or reporting will see a different order than what the UI shows, causing confusion.
- If the exported file is re-imported, the milestone association order may change.
- Downstream tools that compare exported milestone lists with UI-displayed lists will report false differences.

## Notes

- Milestone `Sprint 1` was associated before `Sprint 2`. The UI preserves the association order (Sprint 1 first). The export reverses this, outputting the most recently associated milestone first.
- The separator in the export is `|` (pipe). Since requirement names can also contain `|` characters (confirmed by `REQ-SC02 Pipe|Separator Test`), the separator choice itself is ambiguous — but that is a separate issue.
