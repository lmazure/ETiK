# Bug 005 — RETRACTED

**This report was based on incorrect test analysis and does not describe a real bug.**

## What went wrong in the analysis

The initial analysis claimed that the rich-text XLS export only showed Version 1 of a multi-version requirement, omitting Version 2. This conclusion was wrong because the script used to analyze the file (`/tmp/analyze_comprehensive.js`) contained a hardcoded path to `comprehensive-export.xls` — a file exported *before* Version 2 was created. The `sed` substitution in the command line only replaced text in the stdout, not in the script's source, so the script still read the old file.

When `version2-export.xls` (the export made *after* creating Version 2) is analyzed directly, it correctly contains **9 rows**: Version 1 and Version 2 of SC-001 are both present, and the TC link count is correct for each version.

## Confirmed correct behavior

With "Keep the rich text format" enabled:

| REQ_VERSION_NUM | REQ_VERSION_ID | REQ_VERSION_#_TC | REQ_VERSION_MILESTONE |
|-----------------|----------------|------------------|-----------------------|
| 1 | 5 | 0 | Sprint 2\|Sprint 1 |
| 2 | 14 | 1 | _(empty)_ |

Both versions are exported. TC links are correctly attributed to Version 2. The export format is correct.
