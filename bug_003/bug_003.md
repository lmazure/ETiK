# Bug 003: Typo in export column name `REQ_VERSION_#_ATTACHEMENT`

## Summary

The REQUIREMENT sheet of the exported XLS file contains a column named `REQ_VERSION_#_ATTACHEMENT`, which is a misspelling. The correct English spelling is `ATTACHMENT` (double T). The column is named `REQ_VERSION_#_ATTACHEMENT` in all exports.

## Steps to Reproduce

1. Navigate to any requirement folder in the Requirements workspace.
2. Click the import/export toolbar button and select **Export**.
3. In the dialog, select format **XLS** and click **Export**.
4. Open the downloaded XLS file.
5. Examine the column headers of the **REQUIREMENT** sheet.

## Expected Result

The column header should be `REQ_VERSION_#_ATTACHMENT` (double T, as per correct English spelling of "attachment").

## Actual Result

The column header is `REQ_VERSION_#_ATTACHEMENT` (single T, missing the second T).

## Evidence

Column headers from the exported REQUIREMENT sheet:

```
PROJECT_ID, PROJECT_NAME, REQ_PATH, REQ_NUM, REQ_ID, REQ_VERSION_NUM,
REQ_VERSION_ID, REQ_VERSION_REFERENCE, REQ_VERSION_NAME, REQ_NATURE,
REQ_VERSION_CRITICALITY, REQ_VERSION_CATEGORY, REQ_VERSION_STATUS,
REQ_VERSION_DESCRIPTION, REQ_VERSION_#_TC, REQ_VERSION_#_ATTACHEMENT,  ← typo here
REQ_VERSION_CREATED_ON, REQ_VERSION_CREATED_BY, REQ_VERSION_LAST_MODIFIED_ON,
REQ_VERSION_LAST_MODIFIED_BY
```

## Impact

- **Import compatibility**: If any tool or script parses the exported XLS to generate import files, the column name `REQ_VERSION_#_ATTACHEMENT` would be used in the import file. If the import parser expects the correctly-spelled column name `REQ_VERSION_#_ATTACHMENT`, the import would fail or ignore attachment counts.
- **Documentation inconsistency**: The import specification (if it uses the correct spelling) would not match the export.

## Environment

- SquashTM instance: http://host.docker.internal:8090/squash
- Export format: XLS
- Reproduced with: all exported files (baseline, special-chars, full project exports)
